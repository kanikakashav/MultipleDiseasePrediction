The notebooks in this folder are exploratory training notebooks.

For local project execution, use these files in the project root instead:

- `train_models.py` to retrain and save the models
- `MultipleDiseasePridiction_app.py` to run the Streamlit interface
- `project_config.py` for shared paths, model filenames, and app input definitions

Commands:

```bash
cd /home/thearchfiend/MultipleDiseasePrediction-main/MultipleDiseasePrediction-main/Project
./mdp/bin/python --version
./mdp/bin/pip install -r requirements.txt
./mdp/bin/python train_models.py
./mdp/bin/streamlit run MultipleDiseasePridiction_app.py
```

The Streamlit app will usually open on a local URL like:

```text
http://127.0.0.1:8501
```

Why this matters:

- the notebooks were created for Google Colab and still contain Drive-based paths
- the app now uses a shared local configuration instead of relying on notebook assumptions
- the saved model files in this folder are generated by `train_models.py`
